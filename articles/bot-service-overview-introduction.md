# <a name="about-azure-bot-service"></a>Общие сведения о службе Azure Bot

[!INCLUDE [pre-release-label](includes/pre-release-label-v3.md)]

Служба Azure Bot предоставляет средства для сборки, тестирования, развертывания и управления интеллектуальными ботами в одной среде. В модульной и расширяемой платформе, предоставленной пакетом SDK, разработчики могут использовать шаблоны для создания ботов, которые обеспечивают речь, понимание языка, вопросы, ответы и многое другое.  

## <a name="what-is-a-bot"></a>Что такое бот?
Бот — это приложение, с которым пользователи взаимодействуют, общаясь с помощью текста, графики (карт) или речи. Это может быть простой диалог "вопрос — ответ" или сложный бот, который позволяет людям взаимодействовать со службами разумным образом, используя сопоставление шаблонов, отслеживание состояния и методы искусственного интеллекта, которые хорошо интегрированы с существующими бизнес-службами. Ознакомьтесь с [примерами использования](https://azure.microsoft.com/services/bot-service/) ботов.  

## <a name="building-a-bot"></a>Сборка бота 
Вы можете выбирать среду разработки или средства командной строки для создания бота на платформах C# или Node.js. Предоставляются средства для различных этапов разработки бота. Используйте их, чтобы начать сборку своего бота.    

![Обзор бота](media/bot-service-overview.png) 

## <a name="plan"></a>План 
Прежде чем писать код, просмотрите [руководство по разработке](bot-service-design-principles.md)  бота, чтобы ознакомиться с рекомендациями и определить потребности своего бота. Можно создать простой бот или с более сложными возможностями, такими как речь, понимание языка, вопросы и ответы, или возможностями для извлечения знаний из различных источников и предоставления интеллектуальных ответов.  

> [!TIP] 
>
> Установите нужный шаблон.
>  - Пакет SDK Bot Builder версии 3 для .NET [шаблон VSIX](https://marketplace.visualstudio.com/items?itemName=BotBuilder.BotBuilderV3) 
>  - Пакет SDK Bot Builder версии 3 для Node.js [шаблон Yeoman](https://www.npmjs.com/package/generator-botbuilder) 
>
> Установите средства.
> - Загрузите [средства CLI](https://github.com/Microsoft/botbuilder-tools) для создания и управления активами бота. Эти средства помогают управлять из командной строки файлом конфигурации бота, приложением LUIS, базой знаний QnA и многим другим. Дополнительные сведения см. в файле [readme](https://github.com/Microsoft/botbuilder-tools/blob/master/README.md).
> - [Эмулятор](https://github.com/Microsoft/BotFramework-Emulator/releases) для тестирования бота.
>
> При необходимости используйте следующие компоненты бота.  
> - [LUIS](https://www.luis.ai/), чтобы добавлять в бот распознавание речи.
> - [QnA Maker](https://qnamaker.ai/), чтобы отвечать на вопросы пользователя используя более естественный способ общения.
> - [API распознавания речи Bing](https://azure.microsoft.com/services/cognitive-services/speech/), чтобы конвертировать аудио в текст, понимать намерение и преобразовывать текст в речь.  
> - [API проверки орфографии Bing](https://azure.microsoft.com/services/cognitive-services/spell-check/), чтобы исправлять орфографические ошибки, распознавать имена, торговые марки и сленг. 
> - [Cognitive Services](bot-service-concept-intelligence.md) для различных интеллектуальных компонентов. 


## <a name="build-your-bot"></a>Сборка бота 
Бот — это веб-служба, которая реализует интерфейс для общения, а также взаимодействует со службой Azure Bot. Это решение можно создать в любом количестве сред и языков. Мы предлагаем средства для легкого запуска в Visual Studio или Yeoman, или непосредственно на портале Azure. Ниже приведены некоторые средства и службы, которые можно использовать.

> [!TIP]
>
> Создайте бот, используя [пакет SDK](~/dotnet/bot-builder-dotnet-quickstart.md), [портал Azure](bot-service-quickstart.md) или [средства CLI](~/bot-builder-create-templates.md).
>
> Добавление компонентов. 
> - Добавьте модель распознавание речи [LUIS](https://docs.microsoft.com/en-us/azure/cognitive-services/luis/home). 
> - Добавьте базу знаний [QnA Maker](https://docs.microsoft.com/en-us/azure/cognitive-services/qnamaker/home), чтобы ответить на заданные пользователями вопросы.  
> - Улучшите взаимодействие с пользователем с помощью карт, распознавания речи или перевода. 
> - Добавьте логику в бот с помощью пакета SDK для Bot Builder.   

## <a name="test-your-bot"></a>Тестирование бота 
Боты — это сложные приложения, где работают вместе много различных частей. Как и любое другое сложное приложение, бот может иметь некоторые нестандартные ошибки или он может вести себя иначе, чем ожидалось. Поэтому перед публикацией протестируйте бот.

> [!TIP] 
>
> - [Проверить бот с помощью эмулятора](bot-service-debug-emulator.md)
> - [Проверить бот в веб-чате](bot-service-manage-test-webchat.md)

## <a name="publish"></a>Опубликовать 
Когда все готово, опубликуйте бот в Azure или в собственной веб-службе, или в центре обработки данных. Вы можете настроить непрерывное развертывание, которое позволяет развертывать бот локально. Это удобно, если необходимо проверить бот в системе управления версиями, например GitHub или Visual Studio Team Services. После записи изменений в исходном репозитории они будут автоматически развернуты в Azure.

> [!Tip]
>
> - [Развертывание в Azure](bot-service-build-continuous-deployment.md)

## <a name="connect"></a>Подключение          
Подключите бот к каналам, таким как Facebook, Messenger, Kik, Skype, Slack, Microsoft Teams, Telegram, текстовым и SMS-сообщениям, Twilio, Кортана и Skype, чтобы увеличить взаимодействие и привлечь больше клиентов.  
  
> [!TIP]
>
> - [Выбор каналов для добавления](bot-service-manage-channels.md)


## <a name="evaluate"></a>Evaluate 
Используйте данные, собранные на портале Azure, чтобы определить возможности для повышения производительности бота. Вы можете получить данные уровня службы и инструментирования, например трафик, задержку и интеграцию. Аналитика также поддерживает отчеты уровня общения для данных пользователя, сообщений и каналов.

> [!Tip]
>
> - [Сбор аналитики](bot-service-manage-analytics.md)

# <a name="about-azure-bot-service"></a>Общие сведения о службе Azure Bot

[!INCLUDE [pre-release-label](includes/pre-release-label.md)]

Служба Azure Bot предоставляет средства для сборки, тестирования, развертывания и управления интеллектуальными ботами в одной среде. В модульной и расширяемой платформе, предоставляемой пакетом SDK, разработчики могут использовать шаблоны для создания ботов, которые обеспечивают распознавание речи, понимание естественного языка, обработку вопросов и ответов и многое другое.  

## <a name="what-is-a-bot"></a>Что такое бот?

Бот — это приложение, которое взаимодействует с людьми с помощью диалогов. Программное обеспечение может взаимодействовать через текст, речь, изображения или меню и выполнять задачи, связанные с ведением диалога. Этот может быть простой бот для обмена вопросами и ответами или сложный бот, с помощью которого пользователи могут взаимодействовать со службами естественным образом. При этом в фоновом режиме, бот использует интеллектуальные методы, тесно интегрированные с существующими службами.

Боты основаны на системах, которые собирают информацию или обеспечивают взаимодействие с пользователями, больше похожее на общение, чем на работу с компьютером. Также они позволяют поручать системам (или интеграции с другими системами) простые задачи, такие как резервирование столика на ужин или сбор сведений из профилей пользователей, когда прямое участие пользователя не является обязательным. 

## <a name="building-a-bot"></a>Сборка бота 

Вы можете выбирать среду разработки или средства командной строки для создания бота на платформах C#, JavaScript, Java или Python. Предоставляются средства для различных этапов разработки бота. Используйте их, чтобы начать сборку своего бота.

Вы можете создать бот как с простыми, так и более сложными возможностями, например распознаванием речи или естественного языка либо с возможностью отвечать на заданные пользователем вопросы.
